# Razorpay Payment Integration Setup

This guide explains how to set up secure Razorpay payment integration for MuseHaus using Vercel Serverless Functions.

## Overview

The payment flow now uses secure server-side order creation and verification:
1. Frontend requests order creation from `/api/create-order`
2. Server creates Razorpay order with validated amount
3. Frontend opens Razorpay checkout with server-generated order_id
4. After payment, frontend sends payment details to `/api/verify-payment`
5. Server verifies payment signature using HMAC SHA256
6. Only if verified, enrollment is created in database

## Getting Razorpay Credentials

### 1. Get Key ID (Frontend)
- Go to https://dashboard.razorpay.com/app/keys
- Sign up/login to Razorpay
- Copy your **Key ID** from Test Mode or Live Mode section
- Test keys start with `rzp_test_...`
- Live keys start with `rzp_live_...`

### 2. Get Key Secret (Backend - CRITICAL)
- On the same page at https://dashboard.razorpay.com/app/keys
- Click the "Reveal" button next to Key Secret
- Copy the **Key Secret** (starts with a long alphanumeric string)
- **WARNING**: Never commit this to git, never expose in frontend code

## Local Development Setup

### Step 1: Update your local .env file

Add these variables to your `.env` file (create it if it doesn't exist):

```bash
# Frontend (public) - can be committed
VITE_RAZORPAY_KEY_ID=rzp_test_YOUR_KEY_ID_HERE

# Backend (server-only) - NEVER commit these
RAZORPAY_KEY_ID=rzp_test_YOUR_KEY_ID_HERE
RAZORPAY_KEY_SECRET=your_key_secret_here
```

### Step 2: Install Vercel CLI (for local testing)

```bash
npm install -g vercel
```

### Step 3: Test serverless functions locally

Instead of `npm run dev`, use:

```bash
vercel dev
```

This will:
- Start the Vite dev server
- Run serverless functions locally at `http://localhost:3000/api/*`
- Allow you to test the full payment flow with backend verification

### Step 4: Test the payment flow

1. Navigate to `http://localhost:3000/workshops`
2. Click "Book — ₹499" or "Combo (2) — ₹799"
3. Login if required
4. Click "Pay via Razorpay"
5. Complete test payment (use test card: `4111 1111 1111 1111`)
6. Payment will be verified server-side
7. Enrollment will be created only if verification succeeds

## Production Deployment (Vercel)

### Step 1: Add environment variables in Vercel

1. Go to your Vercel project dashboard
2. Navigate to **Settings → Environment Variables**
3. Add the following variables:

| Name | Value | Environment |
|------|-------|-------------|
| `VITE_RAZORPAY_KEY_ID` | `rzp_live_YOUR_LIVE_KEY_ID` | Production, Preview |
| `RAZORPAY_KEY_ID` | `rzp_live_YOUR_LIVE_KEY_ID` | Production, Preview |
| `RAZORPAY_KEY_SECRET` | `your_live_key_secret_here` | Production, Preview |

**IMPORTANT:**
- Use LIVE keys for production
- Use TEST keys for preview deployments if you want to test
- Never use `VITE_` prefix for `RAZORPAY_KEY_SECRET` - it's server-only

### Step 2: Deploy to Vercel

```bash
git add .
git commit -m "Add secure Razorpay backend integration"
git push origin master:main --force
```

Vercel will automatically:
- Build the frontend
- Deploy serverless functions
- Apply environment variables
- Make the site live at musehaus.vercel.app

### Step 3: Verify deployment

1. Visit your Vercel dashboard
2. Check that the build succeeded
3. Test the payment flow on the live site
4. Check Vercel function logs for any errors

## Security Notes

### Why Server-Side Order Creation?

**Without server-side order creation (old method):**
- Amount sent from frontend can be tampered with
- User could change ₹499 to ₹1 in browser DevTools
- No way to verify payment authenticity

**With server-side order creation (new method):**
- Amount validated server-side based on workshop
- Order ID generated by Razorpay server
- Payment signature verified using HMAC SHA256
- Tampered payments are rejected

### Environment Variable Security

**Frontend variables (VITE_ prefix):**
- Exposed in browser bundle
- Safe for public keys like Key ID
- NEVER use for secrets

**Backend variables (no VITE_ prefix):**
- Only accessible in serverless functions
- Never exposed to frontend
- Required for Key Secret and other sensitive data

### Payment Verification Flow

1. Razorpay sends `razorpay_order_id`, `razorpay_payment_id`, `razorpay_signature`
2. Server generates HMAC SHA256: `sha256(order_id + "|" + payment_id)` using Key Secret
3. Compares generated signature with received signature
4. If they match: payment is genuine
5. If they don't match: payment was tampered with or fake

## Troubleshooting

### Payment verification fails

**Check:**
- `RAZORPAY_KEY_SECRET` is set correctly in Vercel
- No `VITE_` prefix on the secret variable
- Key Secret matches the Key ID (same mode: test or live)

### Order creation fails

**Check:**
- `RAZORPAY_KEY_ID` is set correctly in Vercel
- Workshop title matches exactly what's in `api/create-order.js`
- Option is either "solo" or "combo"

### Functions not working locally

**Check:**
- Using `vercel dev` instead of `npm run dev`
- `.env` file has all required variables
- Vercel CLI is installed: `npm install -g vercel`

### Mixed content errors (HTTPS/HTTP)

**Check:**
- Vercel automatically uses HTTPS
- Local development with `vercel dev` uses HTTP
- This is normal and won't affect production

## Database Schema Updates

The `enrollments` table now includes:
- `razorpay_payment_id` - Payment ID from Razorpay
- `razorpay_order_id` - Order ID from Razorpay
- `payment_verified` - Boolean flag for verification status

If you have an existing database, run these ALTER statements:

```sql
ALTER TABLE enrollments ADD COLUMN IF NOT EXISTS razorpay_payment_id TEXT;
ALTER TABLE enrollments ADD COLUMN IF NOT EXISTS razorpay_order_id TEXT;
ALTER TABLE enrollments ADD COLUMN IF NOT EXISTS payment_verified BOOLEAN DEFAULT false;
```
